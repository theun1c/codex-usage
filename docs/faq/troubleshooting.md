# FAQ: частые ошибки и отладка

## 1) Агент не видит файлы репозитория
**Симптом:** команда чтения возвращает пустой список или ошибки доступа.

**Проверка:**
```bash
pwd
git rev-parse --show-toplevel
```

**Решение:** запустите агента из корня нужного репозитория и проверьте режим sandbox.

## 2) Не запускаются проверки/тесты
**Симптом:** `command not found` для linters/tests.

**Решение для Arch Linux:**
```bash
sudo pacman -Syu --needed base-devel git nodejs npm python python-pip
```

Если проект на Node.js:
```bash
sudo pacman -S --needed pnpm
```

Если проект на Python:
```bash
python -m venv .venv
source .venv/bin/activate
pip install -U pip
```

## 3) MCP-сервер не подключается
**Симптом:** нет ресурсов, ошибки auth.

**Что проверить:**
- корректность токена/секрета;
- доступность команды MCP-сервера в PATH;
- сетевой доступ (если сервер удалённый).

## 4) Агент внёс изменения, но тесты падают
**Подход:**
1. Запустить тесты локально и снять первый stack trace.
2. Сузить проблему до минимального изменённого файла.
3. Попросить агента сделать targeted-fix по конкретной ошибке.

## 5) Конфликтующие инструкции в нескольких AGENTS.md
**Правило:** более глубокий `AGENTS.md` (в вложенной директории) имеет приоритет в своей области.
